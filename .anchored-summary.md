## Goal
- Automatically export CurseForge ZIP and Modrinth mrpack, publishing both to the Modpacks release on every push to main, with optional Modrinth upload on tagged releases.

## Constraints & Preferences
- CFWidget-based script adds `[update.curseforge]` blocks at CI runtime only — never committed to repo.
- `packwiz cf export` references mods by CF project/file ID; mods without CF metadata (not on CF, version mismatch, or custom mods) get JARs bundled as overrides.
- CurseForge API key is unusable (403 on every endpoint).
- FTB Library (ARR license, CF-only) must be in the CF pack but excluded from the Modrinth mrpack — the auto-updater (packwiz-installer, fetches live pack from GitHub Pages) downloads it on first launch.
- Sodium Extra pinned at 0.6.0 (reverted from 0.8.7).
- NERB and KubeJS Create pinned to prevent version drift with their HARDCODED CF file IDs.
